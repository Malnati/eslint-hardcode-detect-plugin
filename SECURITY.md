# Security Policy

## Supported versions

Security fixes are prioritized for the latest published version of `eslint-plugin-hardcode-detect`.

## Reporting a vulnerability

Please do not open public issues for suspected vulnerabilities.

Use one of these private channels:

- Open a private security advisory on GitHub for this repository.
- If private advisory is not available, open a minimal issue without exploit details and request a private contact channel.

When reporting, include:

- A clear description of the issue.
- A minimal reproduction.
- Potential impact and affected versions.
- Any suggested mitigation.

## Response expectations

- Initial triage target: within 5 business days.
- Status updates: at least once per week while the issue is under active investigation.
- Disclosure: coordinated disclosure after a fix is available.

## Safe handling expectations

- Do not commit real secrets in reproductions.
- Use synthetic data and redact sensitive environment details.
- Follow the project policy in [`specs/agent-integration-testing-policy.md`](specs/agent-integration-testing-policy.md).
