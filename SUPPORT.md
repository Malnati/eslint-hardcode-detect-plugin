# Support

## Where to ask for help

- **Usage questions**: open a discussion or issue describing your ESLint setup and expected behavior.
- **Bug reports**: use the bug template at [`.github/ISSUE_TEMPLATE/bug_report.yml`](.github/ISSUE_TEMPLATE/bug_report.yml).
- **Feature requests**: use [`.github/ISSUE_TEMPLATE/feature_request.yml`](.github/ISSUE_TEMPLATE/feature_request.yml).
- **Documentation feedback**: use [`.github/ISSUE_TEMPLATE/docs_improvement.yml`](.github/ISSUE_TEMPLATE/docs_improvement.yml).
- **Security concerns**: follow [`SECURITY.md`](SECURITY.md) and avoid public exploit details.

## What to include in support requests

- Node.js and ESLint versions.
- Plugin version (`eslint-plugin-hardcode-detect`).
- Minimal config (`eslint.config.*`) and sample file.
- Current behavior, expected behavior, and full lint output.

## Response model

The project is community-maintained. Maintainers prioritize:

1. Security and correctness issues.
2. Reproducible bugs affecting many users.
3. Developer-experience and documentation improvements.
